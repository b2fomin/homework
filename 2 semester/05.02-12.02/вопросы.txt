II) Зачем в проектах используются системы контроля версий?
Система контроля версий позволяет хранить несколько вариантов одного и того
же документа, при необходимости возвращаться к более ранним версиям, определять, 
кто и когда сделал то или иное изменение и т.д.

III) Из каких основных действий состоит взаимодействие с Git?
Проверка состояние репозитория через git status
Индексация внесенных изменений с помощью git add
Коммит изменений с помощью git commit
Создание веток, объединение и переключение между ними 
Откат к старым версиям с помощью git reset

IV) Когда следует создавать отдельные ветки для разработки?
Когда программисти работает над новым дефектом или функционалом.

V) Какие основные элементы содержатся в библиотеке chrono?
duration --- тип данных, значение которого не зависит от системы единиц.
time_point --- тип данных для измерения времени с помощью таймера (system clock
или steady clock).
